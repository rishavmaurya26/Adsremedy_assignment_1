# Employee Data Pipeline

An end-to-end data pipeline built with **Apache Spark 4.0**, **PostgreSQL 15**, and **Docker**.
It reads raw employee CSV data, cleans and transforms it using PySpark, and loads the result into PostgreSQL.

## 1. Project Overview

This pipeline solves the following tasks:

| Task | Description |
|------|-------------|
| Task 1 | Docker environment with Spark + PostgreSQL |
| Task 2 | Generate 1,100 raw employee records with data quality issues |
| Task 3 | PySpark job to clean, transform, and enrich the data |
| Task 4 | Load cleaned data into PostgreSQL via JDBC |

## 6. Installations Done Inside Docker

The following were needed inside the containers to make the pipeline work:


The `docker-compose.yml` then mounts it at:
```
./jars  →  /opt/spark/jars/extra   (inside the container)
```

### Inside `postgres` container

PostgreSQL is fully self-configured via `docker-compose.yml` environment variables.
No manual installation needed. The `init.sql` file is mounted at:
```
/docker-entrypoint-initdb.d/init.sql
```
PostgreSQL automatically runs any `.sql` file in that folder on first boot,
which creates the `employees_clean` table and its indexes.

## 8. How the Pipeline Works

The main file is `scripts/spark_pipeline.py`. It runs 6 stages:

### Stage 1 — Read Raw CSV
Reads `employees_raw.csv` with `multiLine=true` so quoted salaries like `"$75,000"` parse correctly. All columns read as strings initially.

### Stage 2 — Data Quality Report (Task 3.1)
Logs counts before any changes so you have a before/after picture:
- Null/empty employee_id count
- Duplicate employee_id count
- Invalid email count (missing `@` or `.`)
- Future hire_date count

### Stage 3 — Clean and Transform (Task 3.2)

| Field | What was wrong | Fix applied |
|-------|---------------|-------------|
| `first_name` / `last_name` | Mixed case e.g. `JOHN`, `doe` | `initcap()` → Proper Case |
| `email` | Uppercase, missing `@`, no TLD | `lower()` + regex validation; invalid rows dropped |
| `salary` | `"$75,000"` format | Strip `$`, `,`, `"` → cast to `DECIMAL(10,2)` |
| `hire_date` | Some dates in the future | Future dates set to null → rows dropped |
| `birth_date` | String format | Parsed to `DATE` |
| `age` | Not in raw data | Calculated: `datediff(today, birth_date) / 365.25` |
| `tenure_years` | Not in raw data | Calculated: `months_between(today, hire_date) / 12` |
| `salary_band` | Not in raw data | Junior < 50k / Mid 50–80k / Senior > 80k |
| `status` | `ACTIVE`, `active` mixed | `initcap()` → normalised to `Active` |
| `state` | Mixed case | `upper()` → always 2-letter uppercase |
| `manager_id` | Empty strings | Empty → null, cast to `INTEGER` |

### Stage 4 — Data Enrichment (Task 3.3)

| New Column | Logic |
|------------|-------|
| `full_name` | `first_name + " " + last_name` |
| `email_domain` | Split email on `@`, take the right part |

### Stage 5 — Load to PostgreSQL (Task 4)
Writes cleaned data to `employees_clean` table using JDBC with `mode=append`.
Includes error handling and logging.

### Stage 6 — Post-load Verification
Reads back from the database and prints:
- Total row count
- Department distribution
- Salary band breakdown

## 9. Executing the Pipeline

### Run the Spark job

```bash
docker exec spark_master \
  /opt/spark/bin/spark-submit \
    --master spark://spark-master:7077 \
    --jars /opt/spark/jars/extra/postgresql-42.7.3.jar \
    /opt/scripts/spark_pipeline.py
```

> **Important:** Always use `/opt/spark/bin/spark-submit` — NOT just `spark-submit`.
> The container does not have Spark binaries in `$PATH`.

## 10. Verifying the Output

### Option A — Interactive PostgreSQL terminal

```bash
# Connect to the database
docker exec -it employee_postgres psql -U sparkuser -d employeedb
```

Once inside, run:

```sql
-- Total rows loaded
SELECT COUNT(*) FROM employees_clean;

-- Sample 5 rows
SELECT employee_id, full_name, email, department, salary, salary_band
FROM employees_clean LIMIT 5;

-- Salary band breakdown
SELECT salary_band, COUNT(*) as total
FROM employees_clean
GROUP BY salary_band
ORDER BY total DESC;

-- Department breakdown
SELECT department, COUNT(*) as total
FROM employees_clean
GROUP BY department
ORDER BY total DESC;

-- Confirm no future hire dates (should return 0)
SELECT COUNT(*) FROM employees_clean
WHERE hire_date > CURRENT_DATE;

-- Confirm no invalid emails (should return 0)
SELECT COUNT(*) FROM employees_clean
WHERE email NOT LIKE '%@%.%';

-- Exit
\q
```

### Option B — Run the verify script

```bash
# Copy script into container and run
docker cp scripts/verify_and_analyse.sql employee_postgres:/tmp/verify.sql

docker exec -it employee_postgres \
  psql -U sparkuser -d employeedb -f /tmp/verify.sql
```
